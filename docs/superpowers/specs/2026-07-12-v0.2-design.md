# wc2026-sim v0.2 — Design

Date: 2026-07-12 (real tournament is at the semifinal stage)

## Goal

Make the deployed simulator self-updating and more informative now that the
real 2026 World Cup is deep into the knockout rounds, then ship to Railway.

## Scope

### 1. Knockout elimination tracking (finish in-flight work)

`World.knockout_out: HashSet<usize>` — teams that lost a recorded real
knockout match. `ko_winner` short-circuits so an eliminated team can never
win a simulated hypothetical match. `update_from_live` derives the set and
logs dropped/unrecognized scrape rows instead of silently skipping.
Already implemented and tested in `src/sim.rs`; this release commits it.

### 2. Automatic background live-data refresh (backend)

Today state only updates when a user POSTs `/api/refresh`. A fresh deploy
serves stale static data from `src/data.rs` (first round of group matches)
until someone clicks refresh.

Design: a `tokio::spawn` background task in `main.rs` that runs
`scraper::fetch_all` + `world.update_from_live` + caches `live_data`
immediately on startup and then every `LIVE_REFRESH_MINUTES` (env,
default 30, `0` disables). Failures log a warning and retry at the next
tick; the server keeps serving the last good state. Reuses the exact same
code path as the `/api/refresh` handler (extracted into a shared helper in
`handlers.rs`).

### 3. Odds module (`src/odds.rs`, currently an empty stub)

Pure functions converting Monte Carlo probabilities to betting-style odds:
- `decimal_odds(p) -> Option<f64>` — fair decimal odds `1/p`, `None` for p≤0
- `format_decimal(p) -> String` — e.g. `"3.45"`, `"—"` when impossible

Exposed as `win_odds: Option<f64>` on `TeamRow` (champion odds per team).
Frontend mirrors the field in `api.ts` and shows odds in the champion
banner and results table.

### 4. Frontend auto-load

On mount: `GET /api/live` to hydrate the (now automatically refreshed)
cached live data, and auto-run the initial 50k-sim forecast. Show
`fetched_at` as a "last updated" hint. The manual buttons stay.

## Non-goals

- Wiring the offline Dixon-Coles model into the live sim (stays offline).
- `src/piratings.rs` stays an empty stub.
- No new state management or UI framework changes.

## Testing

- Unit tests for `odds.rs` (round-trip, edge cases p=0, p=1).
- Existing determinism/count-sum tests must keep passing.
- `npx tsc --noEmit` + `npm run build` for the frontend.
- Local end-to-end smoke test (`/api/health`, `/api/simulate`) before deploy.

## Deploy

Railway project `worldcup-2026-simulator` (Dockerfile build, health check
`/api/health`). Verify `KIMI_API_KEY` is set on the service; deploy with
`railway up`; verify production `/api/health` and `/api/simulate`, and that
startup logs show the initial live refresh.
